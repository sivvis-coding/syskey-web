# syskey-web

> **Bulk file upload · Full-text search · Document tagging** — served from a single CLI command.

`syskey-web` is a desktop/web application that runs entirely from your terminal.
Install it once via `pipx`, run the command, and a browser window opens with the full interface.

---

## Features

| Feature | Description |
|---|---|
| 📁 Bulk upload | Drag & drop any number of files |
| 🔍 Full-text search | Keyword search over file contents (SQLite FTS5) |
| 🏷️ Tag classification | Assign and manage tags per document |
| ✨ Highlight snippets | See the exact sentence where the keyword appears |

---

## Quick start (end-user)

```bash
pipx install syskey-web
syskey-web          # starts server + opens browser automatically
```

Options:

```
syskey-web --help
syskey-web --port 9000 --no-browser
syskey-web --reload   # dev mode with auto-reload
```

## Development setup

### Prerequisites

* Python ≥ 3.10
* Node.js ≥ 18

### 1. Clone the repo

```bash
git clone https://github.com/sivvis-coding/syskey-web.git
cd syskey-web
```

### 2. Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -e "."
```

Start the API server (without the compiled frontend):

```bash
syskey-web --reload
# API docs: http://127.0.0.1:8000/docs
```

### 3. Frontend

```bash
cd frontend
npm install
npm run dev    # Vite dev server on http://localhost:5173 — proxies /api to :8000
```

### 4. Build the frontend and bundle it into the Python package

```bash
cd frontend
npm run build
# Output goes to backend/syskey/static/ (served by FastAPI)
```

After building, `syskey-web` serves the full SPA at `/`.

## API reference

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/api/files/upload` | Upload one or more files (multipart) |
| `GET` | `/api/files/` | List all uploaded files |
| `GET` | `/api/search/?q=<keyword>` | Full-text search with snippets |
| `GET` | `/api/tags/{file_id}` | Get tags for a file |
| `PUT` | `/api/tags/{file_id}` | Replace tags for a file |

Interactive docs available at `http://127.0.0.1:8000/docs` when the server is running.

## Data storage

All data is stored locally in `~/.syskey/`:

```
~/.syskey/
├── syskey.db      # SQLite database (files, tags, FTS5 index)
└── uploads/       # Uploaded file blobs (content-addressed)
```

No external services, no cloud storage, no internet required.

## Publishing to PyPI

1. Build the frontend first:

   ```bash
   cd frontend && npm run build
   ```

2. Build the Python wheel:

   ```bash
   cd backend
   pip install build
   python -m build
   ```

3. Upload to PyPI:

   ```bash
   pip install twine
   twine upload dist/*
   ```

   Or use **Trusted Publisher** (OIDC) via GitHub Actions for a fully automated release.
